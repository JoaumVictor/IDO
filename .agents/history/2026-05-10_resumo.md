# Resumo: Inicialização do Projeto e Banco de Dados
**Data:** 10/05/2026

## 🎯 Objetivo
Dar o primeiro passo para a construção do MVP Alpha do IDO, estruturando o monorepo, inicializando o frontend (Next.js) e definindo o esquema do banco de dados no Supabase.

## ✅ O que foi feito:
- **Banco de Dados (Schema):** Criado e documentado o esquema de tabelas (`profiles`, `ido_stats`, `posts`, `interactions`) no arquivo `/docs/DATABASE_SCHEMA.md`.
- **Estrutura (Monorepo):** Estabelecida a divisão clara entre `/frontend` e `/backend`.
- **Frontend:** Instalado e configurado o projeto base em Next.js (App Router, Tailwind CSS, TypeScript) dentro da pasta `/frontend`.
- **Backend:** Inicializado o projeto Supabase na pasta `/backend` (via `supabase init`), preparando o terreno para as Edge Functions e migrations.

## ⏭️ Contexto para Próximos Passos
- Iniciar a implementação das interfaces no frontend (mobile-first), começando possivelmente pela configuração de layout base (bottom tab) ou tela de login.
- Configurar efetivamente as migrations no Supabase com base no schema que documentamos.
